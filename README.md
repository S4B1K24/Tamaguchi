# Tamaguchi
Запуск:
  1. Запускаем исполняемый файл;
  2. Пишем им нашего питомца;
  3. Выбираем действия.

Обязательные требования к разрабатываемому приложению:
  1.	Должна быть возможность ввести имя питомца.
  2.	Питомец должен обладать следующими свойствами (показателями): 
    1.	уровень здоровья (минимальное значение = 0, максимальное значение = 10, начальное значение = 10);
    2.	уровень голода (минимальное значение = 0, максимальное значение = 10, начальное значение = 0);
    3.	уровень усталости (минимальное значение = 0, максимальное значение = 10, начальное значение = 0);
    4.	уровень счастья (минимальное значение = 0, максимальное значение = 10, начальное значение = 10).
  4.	В приложении, с помощью меню, должны быть доступны следующие действия: 
    1.	покормить;
    2.	поиграть;
    3.	укачать (сон);
    4.  лечение.
  5.	Действия (перечислены в пункте 3), должны влиять на показатели (перечислены в пункте 2) следующим образом: 
    1.	кормление - понижает уровень голода. При избыточном кормлении (переедание) понижается показатель здоровья на 1 пункт.
    2.	игра - повышает уровень усталости и уровень счастья. При избыточной игре понижается показатель здоровья на 1 пункт, а показатель голода увеличивается на 1 пункт.
    3.	укачивание (сон) - снижает полностью уровень усталости, восстанавливает 1 пункт здоровья, увеличивает показатель голода на 1 пункт, уменьшает 1 пункт счастья.
    4. лечение	- восстанавливает 1 пункт здоровья, увеличивает показатель счаться на 1 пункт, увеличивает показатель голода на 1 пункт.
7.	Критическим считается уровень здоровья = 0, при котором питомец заболевает и игра завершается.
